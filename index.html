  • Maven users
  • Shell aficionados

The Maven way

Define your migrations in src/main/resources/changelog.xml:

<?xml version="1.0" encoding="UTF-8"?>
<changelog>
    <changeset id="hello-world" author="you">
        <query><![CDATA[CREATE (n:Sentence {text:'Hello monde!'}) RETURN n]]></query>
    </changeset>
    <changeset id="hello-world-fixed" author="you">
        <query><![CDATA[MATCH (n:Sentence {text:'Hello monde!'}) SET n.text='Hello world!' RETURN n]]></query>
    </changeset>
<changelog>

Then, include Liquigraph plugin in your pom.xml: 

<plugin>
    <groupId>org.liquigraph</groupId>
    <artifactId>liquigraph-maven-plugin</artifactId>
    <version>XXX</version>
    <configuration>
        <changelog>changelog.xml</changelog><!-- classpath location -->
        <jdbcUri>jdbc:neo4j://localhost:7474</jdbcUri>
    </configuration>
</plugin>

And then, simulate the changes:

$> mvn liquigraph:dry-run
# prints queries that would be run
$> less target/output.cypher

And finally, run:

$> mvn liquigraph:run
